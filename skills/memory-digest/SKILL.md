# Memory Digest — 轻量记忆上下文

基于 Markdown 文件的轻量记忆系统，实现 chunk→summary→tree 模式，不依赖 SQLite 或向量数据库。

## 核心理念

每次重要会话结束后提取结构化信息，通过文件链实现跨会话上下文持久化。

## 存储结构

```
workspace-files/.context/memory/
├── YYYY-MM-DD.md        # 每日记忆（会话级 chunk）
├── weekly-YYYY-WXX.md   # 周度滚动摘要
└── INDEX.md             # 最近 7 天记忆索引
```

## 工作流

### 第一步：提取会话记忆

会话结束时，提取以下结构化信息：

1. **关键决策** — 做了什么选择？为什么？影响什么？
2. **数据发现** — 查询了什么？发现了什么数值/趋势？
3. **行动项** — 下一步要做什么？谁做？何时？
4. **上下文锚点** — 当前项目阶段、正在进行的任务、关键约束

写入 `workspace-files/.context/memory/YYYY-MM-DD.md`，格式：

```markdown
# YYYY-MM-DD 会话记忆

## 会话摘要
(一句话概括本次会话)

## 关键决策
- [决策]: (原因) (影响范围)

## 数据发现
- [指标]: (数值) (对比基准) (趋势方向)

## 行动项
- [ ] (任务) (负责人) (截止时间)

## 上下文锚点
- 当前阶段:
- 进行中任务:
- 关键约束:
```

### 第二步：更新索引

每次写入后更新 `INDEX.md`，只保留最近 7 天的条目。

### 第三步：周度摘要（每周日触发）

汇总本周所有日记忆，合并关键决策、汇总数据趋势、清理完成项、标记延期项。

写入 `workspace-files/.context/memory/weekly-YYYY-WXX.md`。

### 第四步：新会话注入

新会话启动时，读取最近 3 天的日记忆 + 本周周度摘要，生成简洁上下文注入。

## 触发时机

| 场景 | 动作 |
|------|------|
| 用户说"总结一下"、"生成记忆" | 立即执行第一步+第二步 |
| 重要任务完成后 | 主动建议生成记忆摘要 |
| 周日/用户要求 | 执行周度摘要 |
| 新会话启动 | 自动检查并注入近期记忆 |

## 质量要求

- 提取必须具体 — 包含实际数值、文件名、决策参数
- 决策必须带原因 — "为什么"比"做了什么"更重要
- 数据必须有基准 — 不能输出孤立数字
- 行动项必须可执行 — 有动词、有对象、有目标状态
