# Amazon 关键词轻量分层

针对某个 Amazon ASIN，对关键词库做轻量、标准化打标。输出运营能直接筛选、复核、用于 SEO/PPC 的标签。

## 输入要求

开始前，向用户确认以下信息。必填项缺失时主动索要；可选项缺失时按回退规则处理。

### 必填

1. `asin`：目标 ASIN。
2. `title`：该 ASIN 的 Amazon 标题或草稿标题。
3. `keyword_library`：待打标的关键词列表。

### 可选（提供时作为权威边界覆盖 title）

- `bullets`：Amazon 五点描述。提供时视为产品功能/规格/支持场景/不支持内容的权威来源，与 `title` 冲突时以 `bullets` 为准。bullets 是"收窄器"——只用来把判断收紧，不用来把判断放宽。
- `description_or_a_plus`：产品描述、A+、图片文案。
- `own_brand`：用户自己的品牌名。
- `competitor_brands`：已知竞品品牌名。
- `category`：产品品类。
- `must_not_claim`：产品不支持、不能宣称的功能/属性/兼容性/场景。

如果 `bullets` 看起来是营销文案（无具体功能/规格/场景），按只有 title 处理；若 title 模糊且 bullets 也未覆盖该意图，相关性应保守判为"不相关"或下降一档。

---

## 核心标签（枚举值固定，禁止新增）

### traffic_label — 搜索意图距离

| 值 | 判定条件 |
|---|---|
| `精准` | 搜索意图直接匹配 ASIN 的产品本体、核心功能或明确支持的使用场景 |
| `流量` | 仍围绕主产品但意图更宽泛：上位品类、泛称、弱限定场景 |
| `广泛` | 非常宽泛、含义模糊、跨品类，或距离明确购买意图较远 |

`traffic_label` 判断的是搜索意图距离，不是搜索量。高搜索量不等于精准。

### relevance_label — 产品承接能力

| 值 | 判定条件 |
|---|---|
| `相关` | 不含品牌名，且 ASIN 主产品可直接承接搜索意图。不能因为关键词包含产品的某个属性词，就判定为可承接。 |
| `品牌词` | 包含任何品牌名（自有/竞品/疑似）。`traffic_label` 仍独立判断。 |
| `不相关` | ASIN 主产品无法合理承接搜索意图。 |

## 关键判断原则

### 相关性以主产品为准

判断相关性以 ASIN 主产品为准，不以"同品类生态"或"搭配使用"为准。
- 配件、耗材、相邻生态词不能因"同场景"标为相关/流量。
- 赠品/附带小配件不构成关键词承接。
- 品牌词不等于不相关，竞品本体词可以是 `精准 + 品牌词`。
- 不要建议把竞品品牌词写入 listing。

### 仅属性匹配不能判为相关

如果关键词只匹配了 ASIN 的某个属性词（如 rechargeable、cordless、portable、pink 等），但没有明确指向主产品本体或核心功能，不得标为 `相关`。属性词匹配是必要条件，不是充分条件——还需验证关键词的整体搜索意图是否指向主产品。

## 五点（bullets）使用规则

当用户提供了 `bullets` 时严格执行以下规则；未提供时按"回退"处理。

### 五点是收窄器，不是扩展器

- 五点用来把"看似相关"的关键词降级（→ 不相关 / 流量 / 广泛）
- 五点没提到的功能/场景，不能因为 title 模糊就升级为"相关"或"精准"

### 从五点提取四类事实

1. 主产品本体词（"是什么"）
2. 明确支持的功能/规格/使用场景（"做什么、什么参数、什么场合"）
3. 明确不支持/排除的情况（"不适用于…"、"不兼容…"、"非…"、"not for…"）
4. 赠品/附带配件清单（"包含/附赠/含 X"、"with X"、"include X"）

### title 与五点冲突时以五点为准

- 如 title 写 "3-in-1 lamp"，但五点只描述美甲灯功能，则按"美甲灯"判断
- 如 title 写 "professional"，但五点显示是入门款，不要据此把高端意图词升为精准

### 赠品/附带配件不构成关键词承接

- 五点里出现"附赠 5 片甲片 / 1 瓶清洁剂"，则"甲片"、"清洁剂"类关键词不能因产品里有这些小赠品而判为相关
- 关键词指向赠品本身（如 `nail tips`、`cleaner`）→ `relevance_label = 不相关`

### 五点显式排除的功能/场景 → 不相关

- 五点说"不适用于脚趾甲" → `toe nail dryer` 类关键词应为 不相关
- 五点说"不兼容凝胶" → `gel curing` 类关键词应为 不相关
- 五点说"不防水" → `waterproof` 类关键词应为 不相关

### 回退（bullets 未提供时）

"精准"判定恢复为"直接指向主产品本体/核心功能/高意图场景 → 精准"，不要求五点支撑。其余规则按只有 title 处理。

## 判断流程

对 `keyword_library` 中的每个关键词，按以下步骤依次判断：

### 第 1 步：识别关键词类型

判断关键词是否为品牌词（包含自有品牌、竞品品牌或疑似品牌名）。如是 → `relevance_label = 品牌词`，但继续判断 `traffic_label`。

### 第 2 步：判断产品承接能力

关键词的搜索意图是否能被 ASIN 主产品直接承接？
- 是主产品本体 → 可承接
- 是主产品核心功能/明确支持的场景 → 可承接
- 是配件、耗材、相邻品类 → 不可承接 → `不相关`
- 是赠品/附带小配件 → 不可承接 → `不相关`
- 五点显式排除的功能/场景 → 不可承接 → `不相关`

如果不可承接，直接标记 `不相关`，`traffic_label` 仍独立判断。

### 第 3 步：判断搜索意图距离（traffic_label）

基于关键词指向与主产品的距离：

- **精准**：直接指向产品本体、核心功能词、高意图购买场景。例如美甲灯 ASIN 的 `uv nail lamp`、`gel curing light`、`nail dryer for gel nails`。
- **流量**：上位品类词（如 `nail lamp`）、泛称（如 `manicure tool`）、弱限定场景。
- **广泛**：跨品类泛词、含义模糊词、距离购买意图远的浏览型搜索。

### 第 4 步：综合输出

每条关键词输出：

```
关键词 | traffic_label | relevance_label | 判断依据（一句话）
```

## 美甲灯类目专用收紧规则

当 ASIN 主产品是 UV/LED Nail Lamp、Nail Dryer、Gel Curing Lamp、美甲固化灯时，必须严格区分：

- **主产品**：UV/LED 美甲固化灯、凝胶甲烘干灯、nail lamp、nail dryer、gel curing light
- **非主产品**：普通灯具、指甲/脚趾甲本身、甲油胶、甲片、美甲套装、美甲耗材、美甲工具

### 仅属性匹配不能判为相关

如果关键词只匹配了 ASIN 的属性词，但没有明确美甲灯/凝胶固化灯意图，不得标为 `相关`。

属性词包括但不限于：rechargeable、cordless、wireless、portable、small、mini、pink、LED、lamp、light、timer、sensor

例如以下词应标为 `广泛 + 不相关`：
- `led lamp`
- `pink lamp`
- `lamp kit`
- `light kit`
- `small cordless lamp`

### 能判为相关的前提

关键词必须明确指向美甲固化/烘干场景，例如：
- `uv nail lamp` → 精准 + 相关
- `gel nail dryer` → 精准 + 相关
- `led nail curing light` → 精准 + 相关
- `nail lamp for gel polish` → 精准 + 相关

仅凭属性词匹配就判为相关是常见错误，需特别注意。

## 输出格式

对 `keyword_library` 中的所有关键词打标完成后，以表格形式输出：

```markdown
| 关键词 | traffic_label | relevance_label | 判断依据 |
|--------|---------------|-----------------|----------|
| uv nail lamp | 精准 | 相关 | 直接指向主产品本体 |
| nail lamp | 流量 | 相关 | 上位品类词，意图较宽 |
| led lamp | 广泛 | 不相关 | 仅匹配属性词，无美甲固化意图 |
| CND nail lamp | 精准 | 品牌词 | 竞品品牌 + 主产品本体 |
```

### 附加汇总

在表格后附简要汇总：
- 总关键词数
- 各标签分布（精准/流量/广泛的 count，相关/品牌词/不相关的 count）
- 组合分布（如 精准+相关、精准+品牌词 等）
- 值得关注的关键词（边界 case 较多、建议运营复核的词）

---

## 边界处理

遇到难以判断的关键词时：
1. 明确标注"边界"及不确定原因
2. 给出倾向性判断但标注需人工复核
3. 优先保守判断——宁可降级也不要过度承诺
