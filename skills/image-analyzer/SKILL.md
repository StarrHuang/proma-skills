# Image Analyzer — 视觉理解代理

本 skill 通过 `model: Qwen3.6-Plus` frontmatter 让 Proma 自动切换到具备视觉能力的模型。你此时已经运行在 Qwen3.6-Plus 上，具备原生图像理解能力。

## 工作流

### Step 1: 确认图片路径

用户可能提供：
- 绝对路径（如 `/Users/cider/screenshot.png`）→ 直接使用
- 相对路径（如 `./demo.png`）→ 拼接当前工作目录
- URL → 先用 `curl -o /tmp/img_analyze_temp` 下载到本地

### Step 2: 读取并描述图片

使用 `Read` 工具直接读取图片文件。Read 工具支持 PNG、JPG、GIF、WebP 等常见格式。

读取后，详细描述图片中的内容：

- **截图/UI**：界面布局、按钮位置、文字内容、交互元素、颜色方案
- **图表**：图表类型、坐标轴、数据趋势、关键数值、图例
- **架构图**：组件关系、数据流向、层次结构、连接方式
- **照片**：场景、物体、人物、颜色、氛围、构图
- **长图/信息图**：从上到下逐段描述，不遗漏任何区块
- **文档/表格**：提取所有可读文字，保持结构

### Step 3: 返回结果

将描述直接呈现给用户。如果是长图或多内容图，按从上到下、从左到右的顺序逐段描述。描述完整后再询问用户是否需要聚焦某个特定区域做深入分析。

## 多图片处理

多张图片逐一读取，每张读完描述一张，最后汇总。

## 注意事项

- 你现在运行在 Qwen3.6-Plus 上，直接读取图片，不要 spawn 子代理
- 只做描述，不做分析或建议
- 如果 Read 返回错误（文件不存在等），检查路径后重试
- 大图（>20MB）建议先用 `sips -Z 2000` 压缩到合理尺寸后再读
